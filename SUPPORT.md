# Support & Communication: Internara Project

This file provides information on how to get help, report issues, and communicate with the Internara community.

---

## 🆘 How to Get Support

Internara is a high-fidelity project, and we prioritize structured, technical communication.

### 1. GitHub Issues (Bugs & Feature Requests)
If you encounter a bug or have a specific technical feature request:
- Search existing [Issues](https://github.com/reasvyn/Internara/issues) to see if it's already being addressed.
- Use our issue templates to provide a clear description and reproduction steps.
- Tag the issue with the relevant domain module (e.g., `Internship`, `Assessment`).

### 2. GitHub Discussions (General Help & Strategy)
For general questions, implementation advice, or architectural discussions:
- Start a thread in [Discussions](https://github.com/reasvyn/Internara/discussions).
- Engage with the community and maintainers to share ideas and strategy.

---

## 🚨 Security Concerns

**Do NOT report security vulnerabilities via public issues.** Please follow the confidential reporting protocol in `SECURITY.md`.

---

## 🛠 Project Lifecycle & Stability

Internara follows a **Semantic Versioning** strategy. For details on versioning, release schedules, and stability guarantees, consult the **[Versioning Policy](versioning-policy.md)**.
