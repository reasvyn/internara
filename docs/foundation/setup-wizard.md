# Setup Wizard тАФ Browser-Based Initial Configuration

> **Last updated:** 2026-07-22 **Changes:** feat тАФ rewrite to developer reference; merge from `docs/guide/02-setup-wizard.md`

## Description

One-time browser-based wizard that provisions the super admin account, school profile, initial
department, and recovery key. Accessed via a signed URL generated by `php artisan setup:install`.
The wizard is locked after completion and cannot be re-run.

---

## Signed URL Lifecycle

```
setup:install тЖТ signed URL generated (TTL: 60 min)
    тЖУ
Browser opens URL тЖТ wizard loads
    тЖУ
User completes Step 5 тЖТ wizard locked, URL invalidated
    тЖУ
URL expired before use тЖТ setup:reset-token (generates new URL)
```

---

## Wizard Flow

### Step 1 тАФ Environment Audit

Automated checks displayed to the user:

| Check | What It Verifies |
| ----- | ---------------- |
| PHP version | >= 8.4.0 |
| Extensions | All required extensions loaded |
| Permissions | `storage/` and `bootstrap/cache/` writable |
| Database | Connection works |
| CLI tools | Composer, npm available |

Items display as green (pass), yellow (warning), or red (fail). Wizard blocks on red items.

### Step 2 тАФ Super Admin Account

| Field | Required | Behavior |
| ----- | -------- | -------- |
| Name | Auto-filled | Hardcoded to `Administrator` тАФ not editable |
| Username | Auto-filled | Hardcoded to `superadmin` тАФ not editable |
| Email | Yes | Login email for super admin |
| Password | Yes | Min 8 chars, uppercase + lowercase + number |
| Confirm Password | Yes | Must match |

The fixed identity ensures one known, non-renameable admin account exists at all times.

### Step 3 тАФ School Information

| Field | Required | Stored In |
| ----- | -------- | --------- |
| School Name | Yes | `settings` table (`school_name` key) |
| Institutional Code | Yes | `settings` table (`institutional_code` key) |
| Email | Yes | `settings` table (`school_email` key) |
| Address | No | `settings` table |
| Phone | No | `settings` table |
| Website | No | `settings` table |
| Principal Name | No | `settings` table |

### Step 4 тАФ Department

| Field | Required | Model |
| ----- | -------- | ----- |
| Department Name | Yes | `Department` |
| Description | No | `Department` |

Creates the first department. Additional departments are added via Admin тЖТ Departments.

### Step 5 тАФ Finalize & Confirm

User confirms all information is correct. On submit:

1. Save school settings
2. Create department
3. Create super admin user (role: `super_admin`)
4. Generate recovery key
5. Lock setup wizard (`SetupState` тЖТ `completed`)

### Step 6 тАФ Recovery Key Display

A 64-character random key is displayed exactly once. The key is:

- Generated via `random_bytes(32)` тЖТ hex-encoded
- Stored as bcrypt hash in the `users` table
- Plaintext saved to `storage/app/private/.recovery-key` (chmod 0600)
- **Never displayed again** after this screen

## CLI Alternatives

When the wizard cannot be used (e.g., headless deployment):

| Action | CLI Command |
| ------ | ----------- |
| Create super admin | `php artisan admin:create` |
| Create department | `php artisan department:create` |
| Set school settings | `php artisan setting:set school_name "..."` |

See [Backup & Recovery](backup-recovery.md) for admin account CLI commands.

## Troubleshooting

| Problem | Cause | Resolution |
| ------- | ----- | ---------- |
| Setup URL 403 | Token invalid or already used | `php artisan setup:reset-token` |
| Setup URL 404 | Wizard already completed | `php artisan setup:install --force` to reset |
| Can't pass Step 1 | Red audit items | Fix listed issues, refresh page |
| Blank page after submit | Check `storage/logs/laravel.log` | Temporarily set `APP_DEBUG=true` |
