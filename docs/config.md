# ⚙️ Configuration Guide

Complete guide to configuring **Internara** for development, staging, and production environments.

---

## Overview

Internara configuration uses Laravel's standard `.env` file approach. Configuration is split into:

- **`.env`** — Environment-specific settings (gitignored)
- **`config/`** — PHP configuration files (committed)
- **`modules/*/config/`** — Module-specific configs

Key configuration areas:
- Application settings (name, URL, timezone, debug)
- Database connection (SQLite, MySQL, PostgreSQL)
- Mail/Email (SMTP, Mailer drivers)
- Caching & sessions
- Queue & job processing
- File storage
- Security & encryption

---

## Quick Start

### 1. Copy Environment Template

```bash
cp .env.example .env
```

### 2. Generate Application Key

```bash
php artisan key:generate
```

### 3. Set Basic Environment

```bash
# .env
APP_NAME=Internara
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000

APP_TIMEZONE=Asia/Jakarta
APP_LOCALE=id

DB_CONNECTION=sqlite
MAIL_MAILER=log
```

### 4. Run Migrations

```bash
php artisan migrate
```

That's it! Application is ready.

## Application Configuration

### Basic Settings

```env
# Application name (shown in UI)
APP_NAME=Internara

# Environment: local, staging, production
APP_ENV=local

# Debug mode (show detailed errors)
# ⚠️ NEVER set to true in production
APP_DEBUG=true

# Application URL (used for links, redirects)
APP_URL=http://localhost:8000

# Application timezone
# Full list: https://www.php.net/manual/en/timezones.php
APP_TIMEZONE=Asia/Jakarta

# Primary locale (for translations)
APP_LOCALE=id

# Fallback locale (if translation not available)
APP_FALLBACK_LOCALE=en

# Faker locale (for seeding/testing)
APP_FAKER_LOCALE=en_EN

# Encryption key (generated by key:generate)
APP_KEY=base64:...
```

## Database Configuration

Internara supports **SQLite, MySQL, PostgreSQL, and MariaDB**.

### SQLite (Development — Default)

```env
DB_CONNECTION=sqlite
DB_DATABASE=database/database.sqlite
```

**Pros**:
- Zero setup (single file)
- Fast for development
- Good for testing (`:memory:`)

**Cons**:
- Single user only
- Not for production
- Limited concurrency

**Setup**:
```bash
# Database created automatically during migration
php artisan migrate
```

### PostgreSQL (Recommended for Production)

```env
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=internara
DB_USERNAME=postgres
DB_PASSWORD=secure_password
DB_CHARSET=utf8
```

**Pros**:
- Excellent for production
- Strong ACID compliance
- Built-in JSON support
- Scalable

**Setup**:
```bash
# 1. Create database and user
sudo -u postgres createdb internara
sudo -u postgres createuser internara_user
sudo -u postgres psql

# In psql:
ALTER USER internara_user WITH PASSWORD 'secure_password';
GRANT ALL PRIVILEGES ON DATABASE internara TO internara_user;

# 2. Run migrations
php artisan migrate
```

### MySQL/MariaDB

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=internara
DB_USERNAME=root
DB_PASSWORD=secure_password
DB_CHARSET=utf8mb4
DB_COLLATION=utf8mb4_unicode_ci
```

**Setup**:
```bash
# 1. Create database and user
mysql -u root -p

# In MySQL:
CREATE DATABASE internara CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'internara_user'@'localhost' IDENTIFIED BY 'secure_password';
GRANT ALL PRIVILEGES ON internara.* TO 'internara_user'@'localhost';
FLUSH PRIVILEGES;

# 2. Run migrations
php artisan migrate
```

### Connection URL (Alternative)

Instead of separate variables, use `DB_URL`:

```env
# PostgreSQL
DB_URL=postgresql://user:password@127.0.0.1:5432/internara

# MySQL
DB_URL=mysql://user:password@127.0.0.1:3306/internara

# SQLite
DB_URL=sqlite://database/database.sqlite
```

## Mail Configuration (SMTP)

### SMTP Setup (Recommended)

The most common and reliable email setup.

#### 1. Gmail (Example)

```env
MAIL_MAILER=smtp
MAIL_SCHEME=tls
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-app-password  # Generate from Google Account Security
MAIL_FROM_ADDRESS=noreply@internara.test
MAIL_FROM_NAME=Internara
```

**Steps**:
1. Enable 2-factor authentication on Google Account
2. Generate App Password: https://myaccount.google.com/apppasswords
3. Use generated password (16 characters)

#### 2. AWS SES

```env
MAIL_MAILER=ses
AWS_ACCESS_KEY_ID=your_key
AWS_SECRET_ACCESS_KEY=your_secret
AWS_DEFAULT_REGION=ap-southeast-1
MAIL_FROM_ADDRESS=noreply@internara.test
MAIL_FROM_NAME=Internara
```

#### 3. SendGrid

```env
MAIL_MAILER=postmark
POSTMARK_TOKEN=your_token
MAIL_FROM_ADDRESS=noreply@internara.test
MAIL_FROM_NAME=Internara
```

#### 4. Mailgun

```env
MAIL_MAILER=mailgun
MAILGUN_DOMAIN=mg.internara.test
MAILGUN_SECRET=your_api_key
MAIL_FROM_ADDRESS=noreply@internara.test
MAIL_FROM_NAME=Internara
```

#### 5. Local Testing (Mailhog)

For development, use **Mailhog** (local mail catcher):

```bash
# Start Mailhog
mailhog

# In .env
MAIL_MAILER=smtp
MAIL_HOST=127.0.0.1
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null

# View emails at http://localhost:8025
```

### Mail Mailer Options

| Mailer | Use Case | Config | Reliability |
| :--- | :--- | :--- | :--- |
| `smtp` | General purpose | Host, port, credentials | Excellent |
| `sendmail` | Server with sendmail | Path to sendmail binary | Good |
| `ses` | AWS hosted | AWS credentials | Excellent |
| `postmark` | Postmark service | API token | Excellent |
| `resend` | Resend service | API token | Excellent |
| `mailgun` | Mailgun service | Domain + token | Excellent |
| `log` | Development | Write to logs | N/A |
| `array` | Testing | Store in memory | N/A |
| `failover` | Redundancy | Multiple mailers | Very good |
| `roundrobin` | Load balance | Multiple mailers | Good |

### Global "From" Address

```env
# Default sender email and name (used in all emails)
MAIL_FROM_ADDRESS=noreply@internara.test
MAIL_FROM_NAME=Internara
```

Override per email:

```php
Mail::send($mailable)
    ->from('admin@internara.test', 'Admin');
```

### Testing Emails

**Log all emails** (development):
```env
MAIL_MAILER=log
MAIL_LOG_CHANNEL=daily
```

Check logs:
```bash
tail -f storage/logs/laravel.log | grep -i mail
```

**Fake emails** (automated testing):
```php
use Illuminate\Support\Facades\Mail;

Mail::fake();

// Send email
Mail::send(new WelcomeEmail($user));

// Assert it was sent
Mail::assertSent(WelcomeEmail::class);
Mail::assertSentTo($user, WelcomeEmail::class);
```

### Recommended Per Environment

| Environment | Store | Speed | Persistence |
| :--- | :--- | :--- | :--- |
| **Development** | `database` | Slow | Yes |
| **Staging** | `redis` | Fast | Yes |
| **Production** | `redis` | Fast | Yes |

## Session Configuration

```env
# Session driver: database, cookie, file, memcached, redis
SESSION_DRIVER=database

# Session lifetime (minutes)
SESSION_LIFETIME=120

# Encrypt session data
SESSION_ENCRYPT=false

# Cookie path and domain
SESSION_PATH=/
SESSION_DOMAIN=null
```

### Development vs. Production

**Development**:
```env
SESSION_DRIVER=database
SESSION_ENCRYPT=false
SESSION_LIFETIME=120
```

**Production**:
```env
SESSION_DRIVER=cookie
SESSION_ENCRYPT=true
SESSION_LIFETIME=60
```

## Queue Configuration

```env
# Queue driver: database, redis, sync
QUEUE_CONNECTION=database

# Queue timeout (seconds)
QUEUE_TIMEOUT=60
```

### Running Queue Workers

```bash
# Run queue worker (long-lived process)
php artisan queue:work

# With concurrency
php artisan queue:work --queue=default,notifications --tries=3

# With timeout
php artisan queue:work --timeout=60

# In daemon mode (Supervisor)
supervisor/internara-worker.conf
```

### Recommended Per Environment

| Environment | Driver | Why |
| :--- | :--- | :--- |
| **Development** | `sync` | Synchronous (instant) |
| **Development (background jobs)** | `database` | Persistent, no extra service |
| **Staging** | `redis` | Fast, scalable |
| **Production** | `redis` | Fast, reliable, scalable |

## File Storage Configuration

```env
# Default filesystem disk
FILESYSTEM_DISK=local

# AWS S3 (if using remote storage)
AWS_ACCESS_KEY_ID=your_key
AWS_SECRET_ACCESS_KEY=your_secret
AWS_DEFAULT_REGION=ap-southeast-1
AWS_BUCKET=internara-uploads
AWS_URL=https://s3.ap-southeast-1.amazonaws.com/internara-uploads
```

### Storage Paths

```
storage/app/
├── public/              # Publicly accessible files
│   ├── avatars/
│   ├── documents/
│   └── media/
└── private/             # Private files (encrypted)
    ├── backups/
    └── logs/
```

### Configuration

```php
// config/filesystems.php
'disks' => [
    'local' => [
        'driver' => 'local',
        'root' => storage_path('app'),
        'url' => env('APP_URL').'/storage',
        'visibility' => 'private',
    ],

    'public' => [
        'driver' => 'local',
        'root' => storage_path('app/public'),
        'url' => env('APP_URL').'/storage',
        'visibility' => 'public',
    ],

    's3' => [
        'driver' => 's3',
        'key' => env('AWS_ACCESS_KEY_ID'),
        'secret' => env('AWS_SECRET_ACCESS_KEY'),
        'region' => env('AWS_DEFAULT_REGION'),
        'bucket' => env('AWS_BUCKET'),
        'url' => env('AWS_URL'),
    ],
]
```

## Logging Configuration

```env
# Log channel: single, daily, stack, syslog, errorlog
LOG_CHANNEL=stack

# Log level: emergency, alert, critical, error, warning, notice, info, debug
LOG_LEVEL=debug

# Deprecations channel
LOG_DEPRECATIONS_CHANNEL=null
```

### Log Levels

| Level | Use |
| :--- | :--- |
| `emergency` | System unusable |
| `alert` | Immediate action required |
| `critical` | Critical condition |
| `error` | Error condition |
| `warning` | Warning condition |
| `notice` | Normal but significant |
| `info` | Informational |
| `debug` | Debug-level messages |

### Log Stacks

```env
# Single file
LOG_CHANNEL=single
# Output: storage/logs/laravel.log

# Daily rotation
LOG_CHANNEL=daily
# Output: storage/logs/laravel-YYYY-MM-DD.log

# Multiple channels
LOG_CHANNEL=stack
LOG_STACK=single,daily
```

## Security Configuration

### Encryption

```env
# Encryption key (generated by key:generate)
```

### HTTPS Enforcement

```env
# Force HTTPS in production
```

```php
    URL::forceScheme('https');
}
```

## Common Configuration Mistakes

### ❌ Database Connection Fails

```
SQLSTATE[HY000] [1045] Access denied for user
```

**Solution**:
- Check `DB_USERNAME` and `DB_PASSWORD`
- Verify user exists: `mysql -u user -p`
- Check `DB_HOST` is correct
- For PostgreSQL, check port `5432`

### ❌ Mail Not Sending

```
Swift_TransportException: Connection could not be established
```

**Solutions**:
- Check `MAIL_HOST`, `MAIL_PORT`
- Verify credentials: `telnet MAIL_HOST MAIL_PORT`
- For Gmail, generate App Password (not account password)
- Check firewall allows SMTP port (usually 587 or 25)

### ❌ Cache Not Working

```
Connection refused: REDIS_HOST:REDIS_PORT
```

**Solution**:
- Start Redis: `redis-server`
- Or switch to `CACHE_STORE=database`

### ❌ Session Not Persisting

**Solution**:
- Check `SESSION_DRIVER=database`
- Run migration: `php artisan migrate`
- Check `sessions` table exists

## Production Checklist

Before deploying to production:

- [ ] `APP_DEBUG=false`
- [ ] `APP_ENV=production`
- [ ] `APP_KEY` generated and set
- [ ] Database configured (PostgreSQL recommended)
- [ ] `MAIL_MAILER=smtp` with real service
- [ ] `CACHE_STORE=redis` with Redis running
- [ ] `SESSION_DRIVER=cookie` with `SESSION_ENCRYPT=true`
- [ ] `QUEUE_CONNECTION=redis` with workers running
- [ ] Filesystem disk set to `s3` or verified `local`
- [ ] `LOG_LEVEL=warning` (not debug)
- [ ] HTTPS enforced
- [ ] CORS configured for your domain
- [ ] Backup strategy in place
- [ ] Monitoring/alerting configured
- [ ] Rate limiting enabled
